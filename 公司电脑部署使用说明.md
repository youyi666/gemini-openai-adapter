# 公司电脑部署使用说明

这份说明是给到公司以后照着操作用的。不要把 `gemini_cookies.local.json`、`adapter_env.local.ps1`、本机用量日志发给别人，也不要提交到公开仓库。

## 最推荐的方式

进入项目目录后，直接双击：

```text
START_HERE.bat
```

然后按菜单顺序操作：

1. 初始化本地配置文件
2. 安装/更新依赖
3. 刷新浏览器 Cookie
4. 启动服务
5. 运行兼容自测

下面的命令步骤是备用手动流程，菜单能跑通时优先用菜单。

## 1. 这套东西是干什么的

它会在本机启动一个 OpenAI 兼容接口：

```text
http://127.0.0.1:8000/v1
```

Cline、Continue 这类 IDE AI 插件可以把它当成 OpenAI 服务使用，实际请求会转到 Gemini 网页逆向接口。

常用地址：

```text
健康检查：http://127.0.0.1:8000/health
模型列表：http://127.0.0.1:8000/v1/models
用量页面：http://127.0.0.1:8000/usage.html
```

## 2. 公司电脑第一次安装

先安装好这些基础软件：

- Git
- Python 3.11 或更新版本
- 一个能正常登录 Gemini 的浏览器

然后在 PowerShell 里执行：

```powershell
cd D:\
mkdir GeminiAPI
cd D:\GeminiAPI
git clone https://github.com/youyi666/gemini-openai-adapter.git
cd gemini-openai-adapter
```

安装依赖：

```powershell
.\install_adapter_dependencies.bat
```

生成本地配置文件：

```powershell
copy adapter_env.example.ps1 adapter_env.local.ps1
copy gemini_cookies.example.json gemini_cookies.local.json
```

## 3. 填 Gemini Cookie

在公司电脑浏览器里登录：

```text
https://gemini.google.com
```

然后打开浏览器开发者工具，找到 Gemini 网站的 Cookies，复制这两个值到 `gemini_cookies.local.json`：

```json
{
  "__Secure-1PSID": "这里填你的值",
  "__Secure-1PSIDTS": "这里填你的值"
}
```

打开文件可以用：

```powershell
notepad gemini_cookies.local.json
```

注意：

- Cookie 是账号凭据，不要发给别人。
- Cookie 过期或失效后，服务会鉴权失败，需要重新复制。
- 这个文件已经被 `.gitignore` 忽略，不要强制提交它。

### Cookie 自动写回

默认配置已经开启：

```powershell
$env:OPENAI_ADAPTER_COOKIE_WRITEBACK = "1"
$env:OPENAI_ADAPTER_COOKIE_WRITEBACK_INTERVAL_SECONDS = "60"
```

含义是：Gemini 上游客户端如果在运行过程中刷新或轮换了 `__Secure-1PSID` / `__Secure-1PSIDTS`，适配器会自动把新值写回 `gemini_cookies.local.json`。写回会在启动时、每隔 60 秒、请求成功后、服务关闭前尝试执行，但只有账号状态已经认证成功时才会真正写入。

但它不能拯救已经失效的登录态。如果 `/health` 显示 `UNAUTHENTICATED`，还是需要先从浏览器重新复制一次最新 Cookie，重启服务后再继续测试 Cline。之后自动写回会尽量减少反复手动更新。

### 固定 Cookie 来源浏览器

为了避免适配器误读日常 Chrome 主账号 Cookie，建议在公司电脑上给适配器单独准备一个浏览器配置档，只登录 Gemini。可以在 `adapter_env.local.ps1` 里指定 Cookie 来源：

```powershell
$env:OPENAI_ADAPTER_COOKIE_BROWSER = "edge"
$env:OPENAI_ADAPTER_COOKIE_PROFILE = "Default"
```

可选值：

- `auto`：自动从可读浏览器里选择，默认值。
- `chrome`：只从 Chrome 读取。
- `edge`：只从 Edge 读取。

`OPENAI_ADAPTER_COOKIE_PROFILE` 默认是 `Default`。如果你新建了专用配置档，需要填浏览器实际目录名，比如 `Profile 1`，不是浏览器界面里显示的中文昵称。改完后重启服务，再打开控制台首页查看 `Cookie 浏览器` 和 `Cookie Profile` 是否符合预期。

## 4. 启动服务

双击或执行：

```powershell
.\start_ai_server.bat
```

正常启动后会看到类似：

```text
Uvicorn running on http://127.0.0.1:8000
Gemini client initialized successfully.
```

不要关闭这个窗口。Cline 调用期间它要一直开着。

## 5. 简单测试

打开健康检查：

```text
http://127.0.0.1:8000/health
```

如果看到：

```json
{"status":"ok","gemini_client":"initialized"}
```

说明服务正常。

如果 `/health` 里看到：

```text
account_status.name = UNAUTHENTICATED
```

说明 Cookie 已经过期或当前浏览器账号和 Cookie 不一致。先更新 `gemini_cookies.local.json`，然后重启服务，再测试 Cline。

也可以用 PowerShell 测试一次对话：

```powershell
$body = @{
  model = "gemini-3-pro"
  stream = $false
  messages = @(
    @{ role = "user"; content = "用一句话说明 Gemini adapter 已连接。" }
  )
} | ConvertTo-Json -Depth 10

Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/v1/chat/completions" `
  -Method Post `
  -ContentType "application/json" `
  -Body $body
```

## 6. 配置 Cline

在 Cline 里选择 OpenAI Compatible。

填写：

```text
API Provider: OpenAI Compatible
Base URL: http://127.0.0.1:8000/v1
API Key: dummy
Model ID: gemini-3-pro
```

如果某个插件要求 Base URL 不带 `/v1`，可以填 `http://127.0.0.1:8000`。当前 adapter 已经同时支持 `/v1/chat/completions` 和 `/chat/completions`。

推荐模型：

```text
gemini-3-pro
```

如果想快一点，可以用：

```text
gemini-3-flash
```

## 7. 查看用量

双击或执行：

```powershell
.\open_usage_dashboard.bat
```

页面地址：

```text
http://127.0.0.1:8000/usage.html
```

这个页面会显示：

- 总请求次数
- token 估算
- 费用估算
- 每日热力图
- 按模型统计
- 按电脑统计
- 最近请求
- 数据来源

这里的费用是按 Gemini API 官方价格做的本地估算，不是 Gemini 网页版真实账单。

## 8. 多电脑同步用量

每台电脑会写自己的共享用量文件：

```text
usage-sync\adapter_usage.<电脑名>.jsonl
```

默认不会提交真实用量日志。如果你确认仓库是私有仓库，并且想用 Git 汇总家里和公司的用量，可以在用完后执行：

```powershell
git add -f usage-sync/adapter_usage.$env:COMPUTERNAME.jsonl
git commit -m "Sync usage log from $env:COMPUTERNAME"
git push
```

另一台电脑想看到合并统计时执行：

```powershell
git pull
.\open_usage_dashboard.bat
```

如果不想手动提交日志，可以把 `OPENAI_ADAPTER_USAGE_SHARED_DIR` 改成 OneDrive 或 SMB 共享目录，等文件自动同步。

配置位置：

```powershell
notepad adapter_env.local.ps1
```

相关配置：

```powershell
$env:OPENAI_ADAPTER_INSTANCE_ID = $env:COMPUTERNAME
$env:OPENAI_ADAPTER_USAGE_SHARED_DIR = Join-Path $PSScriptRoot "usage-sync"
```

## 10. 常见问题

### 端口被占用

如果 `8000` 被占用，改：

```powershell
notepad adapter_env.local.ps1
```

把端口改成：

```powershell
$env:OPENAI_ADAPTER_PORT = "8001"
```

Cline 的 Base URL 也要改成：

```text
http://127.0.0.1:8001/v1
```

### Cookie 鉴权失败

常见日志：

```text
Gemini authentication failed
```

处理方式：

1. 浏览器重新打开 `https://gemini.google.com`
2. 确认账号能正常使用 Gemini
3. 重新复制 `__Secure-1PSID` 和 `__Secure-1PSIDTS`
4. 更新 `gemini_cookies.local.json`
5. 重启 `start_ai_server.bat`

### Cline 一直说无法回答

先确认 `adapter_env.local.ps1` 里有：

```powershell
$env:OPENAI_ADAPTER_PROMPT_MODE = "auto"
```

然后重启服务。

### Cline 明明读了文件，却还是让我粘贴文件

这是 Gemini 没有正确理解 Cline 工具结果时的常见问题。现在适配器有一层本地文件上下文补偿，任务里明确提到的代码文件会由 adapter 只读附加给 Gemini。

确认 `adapter_env.local.ps1` 里有：

```powershell
$env:OPENAI_ADAPTER_LOCAL_FILE_CONTEXT = "1"
$env:OPENAI_ADAPTER_LOCAL_FILE_CONTEXT_MAX_FILES = "3"
$env:OPENAI_ADAPTER_LOCAL_FILE_CONTEXT_MAX_CHARS = "120000"
```

然后重启服务，并新开一个 Cline 任务再试。

安全说明：这个补偿只会读取任务中明确提到的常见代码/文本文件，并默认屏蔽 Cookie、本地环境、用量日志、token、password、secret 等敏感文件。

### Cline 复杂任务一直显示思考中

Codex 的心跳和进度提示是产品层能力，Cline + OpenAI Compatible 接口不一定能显示同样的中间状态。为了减少“完全没动静”的感觉，adapter 默认启用急切流式：

```powershell
$env:OPENAI_ADAPTER_STREAM_EAGER = "1"
$env:OPENAI_ADAPTER_STREAM_PING_SECONDS = "15"
$env:OPENAI_ADAPTER_STREAM_FALLBACK_NON_STREAM = "1"
$env:OPENAI_ADAPTER_STREAM_FALLBACK_MODEL = "gemini-3-flash"
```

含义：

- 请求一开始就返回 SSE 连接，不再等待 Gemini 第一个文本块。
- Gemini 思考期间每隔一段时间发送 SSE ping，避免连接静默。
- 如果 Gemini Pro 流式接口在输出正文前报错，会自动用 `gemini-3-flash` 兜底一次，减少 Cline 任务直接断掉。
- 如果要恢复旧行为，可以设置 `OPENAI_ADAPTER_STREAM_EAGER = "0"`，但复杂任务更容易在 Cline 里看起来没有心跳。

### 控制 Cline 的上下文大小

仓库里已经加入两个 Cline 专用文件：

- `.clineignore`：让 Cline 忽略 Cookie、本地环境文件、用量日志、debug prompt、缓存、构建产物、测试临时文件。
- `.clinerules`：每次任务都会加载的短规则，包含 PowerShell 写法、文件修改边界、敏感文件保护、Git 提交确认规则。

这两个文件不要写成长篇说明书。真正的部署步骤放在本文档里，`.clinerules` 只保留会影响每次任务安全和稳定性的短规则。

### 让 Cline 写文件或局部修改文件

现在 compact prompt 已经补了 Cline 的写入工具提示：

- `write_to_file`：新建文件，或整体重写一个小文件。
- `replace_in_file`：局部修改已有文件，优先用这个。
- `execute_command`：运行命令，用来测试、格式化、检查结果。

建议先用一个小任务测试，例如：

```text
请在 Gemini-API 目录下创建一个 test_cline_write.txt，内容写入 hello adapter，然后读取它确认内容。
```

再测试局部修改：

```text
请读取 Gemini-API/test_cline_write.txt，把 hello adapter 改成 hello gemini adapter，并读取确认。
```

如果确认没问题，再让它改真实项目文件。涉及删除、大范围重构、提交 git 之前，最好让它先说明计划并等待你确认。

额外注意：现在提示词里已经补了一条防串线规则。只有当前任务明确要求写文件或改文件时，Cline 才应该调用写入工具；执行命令、查看 Git 状态、解释代码时不能顺手改文件。写入内容里也不能夹带用户指令、命令文本、XML 标签或聊天上下文。

### Cline 执行 PowerShell 命令时出错

Windows PowerShell 5.x 不支持很多 Linux/macOS 常见写法。现在 compact prompt 已经补了这几条规则：

- 不要用 `&&` 串联命令。
- 用 `;` 或分开执行命令。
- 进入目录优先写成 `Set-Location "Gemini-API"; git status --short`。
- 检查命令是否成功时，看 `$LASTEXITCODE`。
- 如果终端可见输出里有 `fatal:`、`ParserError`、`InvalidEndOfLine` 或 `^C`，就不能说任务成功，要修正命令后重试。

推荐的语法检查命令：

```powershell
Set-Location "Gemini-API"; python -m py_compile openai_adapter_server.py; if ($LASTEXITCODE -eq 0) { Write-Host "Syntax check passed" } else { exit $LASTEXITCODE }
```

如果看到：

```text
fatal: not a git repository
```

通常是因为当前目录在 `D:\GeminiAPI`，真正的 Git 仓库在 `D:\GeminiAPI\Gemini-API` 或 `D:\GeminiAPI\gemini-openai-adapter` 子目录里。先切进真实仓库目录再执行 `git status --short`。现在 `start_ai_server.bat` 会从脚本所在目录启动，不再依赖固定路径。

### PowerShell 的 curl 发送 JSON 出错

优先用 `Invoke-RestMethod` 测试。Windows PowerShell 里的引号和换行容易把 JSON 搞坏。

### 页面费用和 Cline 右上角费用不一致

以 `usage.html` 为准。Cline 右上角费用是它自己按内置模型表算的，不知道我们这个 Gemini adapter 的真实估算规则。
